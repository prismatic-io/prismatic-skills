Nimbus API Reference

Base URL

All API requests are made to https://api.nimbus.io/v1.

Authentication

Nimbus supports two authentication methods. Both are fully supported and offer equivalent access to the API — pick whichever suits your integration.

Option A — OAuth 2.0

Nimbus implements the OAuth 2.0 authorization code grant. Direct users to the authorization endpoint at https://auth.nimbus.io/oauth/authorize and exchange the returned code for tokens at https://auth.nimbus.io/oauth/token. Supported scopes are records.read, records.write, and webhooks.manage. Access tokens expire after 3600 seconds; refresh tokens are issued alongside them.

Option B — API Key

Alternatively, create an API key in the Nimbus dashboard under Settings → API Keys and pass it in the X-Nimbus-Key request header on every call. The raw key value is sent directly (no "Bearer" prefix).

Records

List records — GET /records

Returns a page of records. Pagination is cursor-based: pass cursor (string, optional) and limit (number, optional, default 50). The response includes a next_cursor field.

Get a record — GET /records/{id}

Returns a single record by id.

Create a record — POST /records

Body fields: title (string, required), body (string, optional), tags (array of strings, optional).

Webhooks

Register a webhook with POST /webhooks. Body: url (string, required), events (array of strings, optional). Event types: record.created, record.updated, record.deleted.

Rate limits

240 requests per minute. The X-RateLimit-Remaining header reports what is left.
