Widgetco API Reference

Base URL

All API requests are made to https://api.widgetco.com/v2. There is a separate sandbox host at https://sandbox.widgetco.com/v2 for testing.

Authentication

Widgetco uses an API key. Pass your key in the X-Widget-Key request header on every call. The raw key value is sent directly (there is no "Bearer" prefix). Keys are created in the Widgetco dashboard under Settings → API Keys. OAuth2 is not supported.

Widgets

List widgets — GET /widgets

Returns a page of widgets. Pagination is cursor-based: pass cursor (string, optional) and limit (number, optional, default 25). The response includes a next_cursor field.

Get a widget — GET /widgets/{id}

Returns a single widget by id.

Create a widget — POST /widgets

Body fields: name (string, required), color (string, optional), weight_grams (number, optional).

Widget object

{
  "id": "wgt_123",
  "name": "Sprocket",
  "color": "blue",
  "weight_grams": 42,
  "created_at": "2026-01-15T10:30:00Z"
}

Webhooks

Register a webhook with POST /hooks. Body: target_url (string, required), events (array of strings, optional — empty means all events).

Event types: widget.created, widget.updated, widget.deleted.

Each delivery is signed. Verify the X-Widget-Signature header, which contains an HMAC-SHA256 hex digest of the raw request body computed with your webhook secret.

Rate limits

120 requests per minute. The X-RateLimit-Remaining header reports what is left.
